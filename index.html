TravelAI Solutions - Current → Figma Proposal → Revised Proposal

CONTEXT

Why This Update is Needed

Analysis from the recent Expedia QBR identified that TravelAI's PinnedSRP property cards are missing key information that competitors display. This gap reduces qualified click-through rates and user confidence in booking decisions.

Competitor Feature Gap Analysis

TravelAI Current State

  • ✗ No distance to city center/attractions
  • ✗ Amenities shown as plain text, not icons
  • ✗ Total price only, no nightly rate breakdown
  • ✗ No free cancellation visibility
  • ✗ No price comparison or "best price" indicators
  • ✗ No scarcity/urgency signals

Competitors Have

  • ✓ "2.1 miles to City center" - distance context
  • ✓ Icon-based amenities (Pool, WiFi, Parking, A/C)
  • ✓ "$174/night" with total shown secondary
  • ✓ "Free cancellation" prominently displayed
  • ✓ "Our lowest price" badges (Vrbo, Expedia)
  • ✓ "Reduced price" and urgency messaging

Source: Expedia QBR - PinnedSRP Competitive Analysis

COMPETITOR

Kayak.com Property Cards

Kayak's property cards feature a horizontal card layout with large imagery, clear pricing hierarchy, and comprehensive property information displayed in a scannable format.

Desktop Experience

Mobile Experience (iPhone 14)

Key Features Observed

Visual Elements

  • ✓ Large property image (left side, ~40% of card width)
  • ✓ Rating badge with score (e.g., "8.9 Very good")
  • ✓ Property type indicator ("Rental")
  • ✓ Save and Share buttons overlay on image

Information Hierarchy

  • ✓ Property name and location displayed prominently
  • ✓ Price comparison from multiple booking sites
  • ✓ Partner logos (Trip.com, Hotels.com, Expedia)
  • ✓ "View Deal" CTA button for each partner

Competitive Comparison

GAP Kayak.com vs Current TravelAI

  • ⚠ Kayak shows: Price comparison across multiple booking sites - TravelAI currently doesn't aggregate partner pricing
  • ⚠ Kayak shows: Clear partner attribution (Trip.com, Hotels.com, Expedia logos) - TravelAI lacks this transparency
  • ⚠ Kayak shows: Multiple CTAs per property (one per partner) - TravelAI has single CTA

PARITY Kayak.com vs TravelAI Parity Proposal

  • ✓ Match: Both show clear rating scores with review counts
  • ✓ Match: Both display property type and location information
  • △ Partial: Kayak shows multi-partner pricing; TravelAI Parity shows single-partner nightly rate + total

EXCEED Kayak.com vs TravelAI Exceed Proposal

  • ★ TravelAI Advantage: "Best Price" verification badge - Kayak doesn't verify best price, just aggregates
  • ★ TravelAI Advantage: "Top X%" rating context - Kayak shows raw scores without percentile ranking
  • ★ TravelAI Advantage: Scarcity indicators ("Only 2 left at this price") - Kayak doesn't show urgency signals

OVERVIEW

Competitor Property Card Analysis

Comprehensive analysis of property card designs across major vacation rental platforms, highlighting UX strengths and opportunities for improvement.

Kayak.com

Desktop

Mobile (iPhone 14)

Good UX

  • 1. Multi-partner price comparison in single view
  • 2. Large, prominent property images (~40% card width)
  • 3. Clear rating badges with review counts
  • 4. Transparent partner attribution with logos
  • 5. Horizontal card layout optimized for scanning

Could Improve

  • 1. No "Best Price" verification - just aggregation
  • 2. Missing distance to city center/attractions
  • 3. No scarcity or urgency indicators
  • 4. Lack of amenity icons for quick scanning
  • 5. No rating context (percentile ranking)

Desktop

Mobile (iPhone 14)

Good UX

  • 4. Free cancellation highlighted
  • 5. Clean vertical card design with image carousel

Could Improve

  • 1. No "Best Price" verification
  • 2. Generic "Special offer" lacks specific scarcity
  • 3. Missing percentile rating context
  • 4. No distance to key attractions shown
  • 5. Limited competitive differentiation
